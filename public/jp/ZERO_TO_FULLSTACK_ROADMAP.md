# ゼロからフルスタック開発者へ — 完全な学習ロードマップ

> コンピュータースキルがない人が、Juan's World のようなシステムを構築して理解できるようになるためのガイド。

**所要時間:** 一貫したパートタイム学習で 8〜12 ヶ月（週 10〜15 時間）。  
**総費用:** $0〜$200（以下に挙げるものはすべて優れた無料オプションがあります）。  
**前提条件:** なし。コンピューターの電源を入れるところから始まります。

---

## 哲学: どのように学ぶか

1. **すぐにものを作る。** 「十分知るまで」待たないでください。以下の各セクションには小さなプロジェクトが含まれています。
2. **一貫性が強度に勝る。** 毎日 30 分は、週に 1 回 5 時間よりも優れています。
3. **苦闘が学習である。** 理解せずにコピー＆ペーストしたら、何も学んでいません。バグを 1 時間見つめて修正したら、すべて学んでいます。
4. **学んだことを教える。** 自分自身、ラバーダック、または友人に向かって概念を声に出して説明してください。シンプルに説明できなければ、まだ理解していません。

## フェーズ 1: Web の仕組み（第 5〜7 週）

**目標:** 実際に構築しようとする前に、ウェブサイトが実際に何であるかを理解する。

### 学ぶこと

- **ウェブサイトとは何か？** 常に起動しているコンピューター（サーバー）に保存されたファイル（HTML、CSS、画像）のコレクション。
- **HTML とは何か？** ページの構造/コンテンツ（見出し、段落、画像、リンク）。
- **CSS とは何か？** スタイル（色、フォント、レイアウト、間隔）。
- **ブラウザーの仕事。** HTML と CSS を読み取り、目に見えるページに変換します。
- **開発者ツール。** 任意のウェブページで右クリック → 「検証」。世界中のあらゆるサイトの HTML と CSS を見ることができます。
- **URL とドメイン。** `https://`、`www.`、`.com`、`?search=term` の意味。
- **ホスティング。** ファイルがコンピューターからインターネットにどのように到達するか。

| リソース | 内容 | 費用 | 時間 |
|----------|-----------|------|------|
| [Mozilla MDN — Web Basics](https://developer.mozilla.org/en-US/docs/Learn/Getting_started_with_the_web) | インターネット上で最も信頼されている Web チュートリアル | 無料 | 6〜8 時間 |
| [Internet 101 (Khan Academy)](https://www.khanacademy.org/computing/computer-science/internet-intro) | インターネットの仕組み、パケット、DNS、IP アドレス | 無料 | 3〜4 時間 |

### 深く理解すべき主要な概念

- **HTML タグ:** 山括弧内のテキストの一部、例: `<h1>こんにちは</h1>`。`<h1>` は開始タグ、`</h1>` は終了タグ、「こんにちは」はコンテンツです。
- **CSS セレクター:** HTML 要素をターゲットにする方法。`h1 { color: red; }` はすべての `<h1>` を赤色にします。
- **DOM (Document Object Model):** ブラウザーがウェブページをオブジェクトのツリーとして内部的に表現したもの。「要素を検証」するとき、DOM を見ています。

## フェーズ 2: プログラミングの基礎（第 8〜18 週）

**目標:** プログラマーのように考える方法を学ぶ。これが最も難しいフェーズです。急がないでください。

### 学ぶこと

- **JavaScript。** Web のプログラミング言語。今後数年間使用します。
- **変数。** データの名前付きコンテナ: `let name = "Juan"`
- **データ型。** 文字列（テキスト）、数値、ブール値（true/false）、配列（リスト）、オブジェクト（ラベル付きデータのコレクション）。
- **関数。** 再利用可能なコードブロック: `function greet(name) { return "こんにちは " + name; }`
- **条件文。** 意思決定: `if (age > 18) { ... } else { ... }`
- **ループ。** 繰り返し: `for`、`while`。
- **デバッグ。** エラーメッセージの読み取り、`console.log()` の使用、問題を小さな部分に分解すること。
- **コマンドライン（ターミナル）。** クリックではなくコマンドを入力すること。最初は威圧的ですが、必須です。
- **Git。** コードのスナップショットを保存して、ミスを元に戻したり協力したりできるツール。

### 深く理解すべき主要な概念

- **変数はラベルの付いた箱です。** `let x = 5` は「x」というラベルの付いた箱を作成し、中に 5 を入れます。
- **関数はレシピです。** 材料（パラメータ）を取り、手順に従い、結果（戻り値）を生み出します。
- **配列は番号付きリストです。** `["りんご", "バナナ", "チェリー"]` — 「りんご」はインデックス 0、「バナナ」は 1、「チェリー」は 2 にあります。
- **オブジェクトはラベル付きのコレクションです。** `{ name: "Juan", age: 30 }` — ラベルで値にアクセスします: `person.name` は `"Juan"` を返します。
- **スコープ。** 関数内で宣言された変数は、その関数内でのみ存在します。これにより、コードの異なる部分が誤って互いに干渉するのを防ぎます。

### プロジェクト: Todo リストアプリを構築する（コンソール版）

ブラウザーコンソール（または Node.js）で実行される JavaScript ファイルを作成し、以下を実行できるようにします:
- リストにタスクを追加する
- タスクを完了としてマークする
- タスクを削除する
- すべてのタスクを印刷する

まだビジュアルインターフェイスは構築しないでください。`console.log()` と関数のみを使用してください。ロジックが重要です。例:

```javascript
let tasks = [];

function addTask(title) {
  tasks.push({ title: title, done: false });
}

function markDone(index) {
  tasks[index].done = true;
}

function showTasks() {
  console.log("マイタスク:");
  tasks.forEach((task, i) => {
    console.log(i + ". " + (task.done ? "[x]" : "[ ]") + " " + task.title);
  });
}
```

### プロジェクト: はじめて Git を使用する

1. Git をインストールします。
2. ターミナルで `learning-journey` フォルダーに移動します。
3. `git init` を実行して Git リポジトリに変換します。
4. `git add .` を実行してファイルをステージングします。
5. `git commit -m "マイファーストコミット"` を実行してスナップショットを保存します。
6. [GitHub](https://github.com) でアカウントを作成します。
7. 新しいリポジトリを作成し、コードをプッシュする手順に従います。

これで、コードをインターネットにバックアップしました。これがプロの開発者が 1 日に何十回も行うことです。

### 深く理解すべき主要な概念

- **DOM はツリーです。** すべての HTML 要素はノードです。JavaScript はこのツリーを横断し、ノードを見つけ、変更し、新しいものを追加し、または削除できます。
- **イベントリスナー。** `button.addEventListener("click", handleClick)` — ユーザーがボタンをクリックすると、`handleClick` 関数を実行します。
- **React state。** `const [count, setCount] = useState(0)` — 変更されたときに自動的に表示ページを更新するデータを保存する方法。
- **React コンポーネント。** HTML のような JSX を返す関数。`<Button color="red">クリックしてね</Button>` — 再利用可能、合成可能、予測可能。
- **Props。** 親コンポーネントから子コンポーネントに渡されるデータ。関数のパラメータのようなものですが、UI の部品用です。

## フェーズ 4: バックエンドとデータベース（第 29〜38 週）

**目標:** 誰かがあなたのウェブサイトを訪問したときにサーバーで何が起こるかを理解する。

### 学ぶこと

- **Node.js。** ブラウザーの外、サーバー上で JavaScript を実行する。
- **HTTP。** ブラウザーとサーバーが通信するために使用するプロトコル。メソッド: GET、POST、PUT、DELETE。ステータスコード: 200、404、500。
- **Express.js。** Node.js でサーバー API を構築するための軽量フレームワーク。
- **API（Application Programming Interfaces）。** 異なるソフトウェアが互いに通信する方法。フロントエンドは API を介してバックエンドにデータを要求します。
- **JSON。** ほぼすべての最新の API が使用するデータ形式。
- **データベース。** なぜ必要か、どのように永続的にデータを保存するか。
- **Redis。** インメモリデータベース。高速、シンプル、キャッシュや小さなデータセットに最適。
- **SQL の基本。** SELECT、INSERT、UPDATE、DELETE。専門家である必要はありませんが、概念を理解すべきです。
- **認証。** ユーザーが自分が誰であるかを証明する方法。パスワード、ハッシュ、セッション、Cookie、JWT トークン。

### 深く理解すべき主要な概念

- **リクエスト/レスポンスサイクル。** ブラウザーが*リクエスト*を送信します。サーバーがそれを処理し、*レスポンス*を返します。Web 上のすべての相互作用はこのパターンに従います。
- **ルート。** サーバー上の特定のリクエストを処理する URL パターン。`GET /users` はユーザーのリストを返すかもしれません。`POST /users` は新しいユーザーを作成するかもしれません。
- **ミドルウェア。** リクエストを受信してレスポンスを送信する間に実行される関数。認証の確認、リクエストのログ記録、JSON 本文の解析などができます。
- **パスワードのハッシュ化。** パスワードをプレーンテキストで保存することは決してありません。PBKDF2 や bcrypt のような一方向の数学的関数を通じて実行し、結果を保存します。たとえデータベースが盗まれたとしても、パスワードは役に立ちません。
- **環境変数。** コードに保存されない秘密の設定値（API キー、データベースパスワード）。Git にコミットされない `.env` ファイルに存在します。

### プロジェクト: Todo アプリの REST API を構築する

以下のエンドポイントを持つ Node.js/Express サーバーを作成します:
- `GET /tasks` — すべてのタスクを返す
- `POST /tasks` — 新しいタスクを作成する（本文: `{ title: "..." }`）
- `PATCH /tasks/:id` — タスクを完了としてマークする
- `DELETE /tasks/:id` — タスクを削除する

最初はタスクを JSON ファイルに保存します。次に Redis にアップグレードします。次に、この API と通信する React フロントエンドを構築します。

### プロジェクト: API に認証を追加する

ユーザー登録とログインを実装します:
- `POST /register` — パスワードをハッシュ化し、ユーザーを保存する
- `POST /login` — パスワードを検証し、セッション Cookie を作成する
- `GET /tasks` を保護して、ユーザーが自分自身のタスクのみを見られるようにする

Juan's World とまったく同じパターンを使用します: ハッシュには PBKDF2、セッション Cookie には HMAC-SHA256。

## フェーズ 5: フルスタックと Juan's World スタック（第 39〜46 週）

**目標:** Juan's World と同じ技術を使用して、完全にデプロイ可能なアプリケーションを構築する。

### 学ぶこと

- **Next.js。** フロントエンドページとバックエンド API ルートを 1 つのプロジェクトに結合する React フレームワーク。
- **サーバーレス関数。** 必要なときにのみクラウドで実行されるコード。管理するサーバーはありません。
- **Vercel。** Next.js アプリを 1 クリックでデプロイ。自動 HTTPS、グローバル CDN、ゼロ構成。
- **Upstash Redis。** REST API を備えた管理型 Redis。サーバーレスに最適です。
- **Resend。** アプリケーションからメールを送信する。
- **Markdown 処理。** `gray-matter` と `marked` を使用して Markdown ファイルを HTML に変換する。
- **フロントエンド/バックエンド統合。** React コンポーネントが API ルートから `fetch()` データを取得します。API ルートは Redis から読み取ります。データは完全な円を描いて流れます。

### 深く理解すべき主要な概念

- **Server Components vs Client Components (React/Next.js)。** ページの一部はサーバー上で実行されます（そこでデータベースに直接アクセスできます）。一部はブラウザー内で実行されます（そこでユーザーのクリックに応答できます）。どちらがどちらかを知ることで、多くのバグを防ぐことができます。
- **Next.js の API ルート。** `app/api/.../route.ts` 内のファイルは自動的に HTTP エンドポイントになります。Express は必要ありません。
- **ビルド vs ランタイム。** デプロイ時に、Next.js はアプリを一度「ビルド」します（最適化されたファイルを生成します）。訪問者が到着すると、「ランタイム」がコードを実行します。一部のものはビルド時にのみ機能します。一部はランタイム時にのみ機能します。
- **フォールバック戦略。** プライマリデータベースがダウンした場合はどうなりますか？ Juan's World のパターン — Redis 優先、ファイルシステム第 2、メモリ第 3 — は信頼性のためのプロフェッショナルグレードのアプローチです。

### プロジェクト: ブログプラットフォームを構築する（あなた自身の Juan's World）

以下を持つ Next.js アプリケーションを作成します:
- 最近の投稿を表示するホームページ
- 各投稿のページ（Redis から取得）
- 管理者ログインページ
- 管理者が投稿を作成/編集/削除できる保護されたダッシュボード
- 投稿コンテンツの Markdown サポート
- 外部スクリプト（あなたの「AI エージェント」）が投稿を公開できる API キーシステム
- Vercel にデプロイする

これは Juan's World の簡易版です。これを構築できれば、システムのすべてのコア概念を理解しています。

### プロジェクト: Juan's World を読んで変更する

Juan's World リポジトリをクローンします。以下を試みてください:
1. ローカルで実行します（`npm install`、`npm run dev`）。
2. `public/` に新しい静的ページを追加します。
3. ランダムな引用を返す新しい API ルートを追加します。
4. 日記ページを変更して、別の順序で投稿を表示します。
5. 新しい環境変数を追加し、API ルートで使用します。

これらすべてができれば、もう初心者ではありません。開発者です。

## 推奨週間スケジュール（パートタイム）

週に 10〜15 時間を充てることができる場合、ここに持続可能なリズムがあります:

| 曜日 | アクティビティ | 時間 |
|-----|----------|-------|
| **月曜日** | 理論を読む/見る（新しい概念） | 1 時間 |
| **火曜日** | インタラクティブな演習で練習する | 1 時間 |
| **水曜日** | プロジェクトに取り組む | 1 時間 |
| **木曜日** | ドキュメントやソースコードを読む | 1 時間 |
| **金曜日** | プロジェクトに取り組む + デバッグする | 1 時間 |
| **土曜日** | 深いプロジェクト作業または追いつく | 2〜3 時間 |
| **日曜日** | 休む、または学んだことを復習する | 0〜1 時間 |

**20 分ルール:** バグや概念に 20 分以上行き詰まったら、正確に何が理解できないかを書き留めて助けを求めてください。何時間も壁に頭をぶつけないでください。[Stack Overflow](https://stackoverflow.com)、Discord コミュニティ、または ChatGPT/Claude に概念を説明してもらいましょう。次に説明を閉じて、自分で実装してみてください。

## マインドセット: 難易度曲線

プログラミングを学ぶことは線形ではありません。予想されることは以下の通りです:

- **第 1〜2 ヶ月:** すべてが新しくてエキサイティングです。速く学んでいます。
- **第 3〜4 ヶ月:** 「絶望の谷」。知っていることは、知らないことがどれだけ多いかを知るのに十分です。プロジェクトはより難しく感じます。これは正常です。続けてください。
- **第 5〜6 ヶ月:** 物事がつながり始めます。チュートリアルなしで小さなプロジェクトを構築できます。
- **第 7〜9 ヶ月:** ドキュメントを読んで、独立して物事を把握できます。
- **第 10〜12 ヶ月:** ゼロから完全なアプリケーションを構築してデプロイできます。

成功する人々は最も賢い人々ではありません。最も一貫している人々です。

## いつ仕事に応募し始めるか

目標が開発者としての雇用である場合:

- **ジュニアフロントエンド開発者:** フェーズ 3（フロントエンド）の後。3〜4 の React プロジェクトのポートフォリオが必要です。
- **ジュニアフルスタック開発者:** フェーズ 5 の後。認証、データベース、デプロイメントを備えた 2〜3 のフルスタックプロジェクトのポートフォリオが必要です。
- **雇用者が見たいもの:**
  - クリーンで文書化されたコードの GitHub プロフィール
  - プロジェクトをホストする個人ウェブサイト
  - 実際の問題を解決するプロジェクト（チュートリアルのコピーではない）
  - 学習できることの証拠（結局のところ、自分自身で学びました）

「準備ができた」と感じる前に応募し始めてください。面接プロセス自体が学習の一部です。
